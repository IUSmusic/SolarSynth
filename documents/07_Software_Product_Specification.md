# Software Product Specification

## Purpose

The software product version is the fuller, more capable form of the project. It should preserve the strengths of the demo while expanding control depth, stability, data support, and professional usability.

## Product direction

The fuller product should be developed as a serious instrument suitable for composition, performance, sound design, and research presentation.

## Recommended deployment targets

- standalone desktop application
- plug-in format support in later phases
- cross-platform development path where practical

## Core feature set

### Instrument structure

- multiple synth tracks
- one or more sequencer tracks
- master bus
- scene management
- preset management
- advanced routing

### Source system

- Helioseismology engine
- Solar Radio engine
- Plasma engine
- Hybrid presentation
- richer source library
- stronger metadata handling

### Touch system

- small track pad
- selected-track large pad
- morph nodes
- hold
- lock
- automation recording
- optional advanced gesture mapping

### Performance system

- MIDI learn
- expression assignment
- pedal support
- scene switching
- live mode
- sound bank mode

### Monitoring

- scope
- spectrum
- source activity
- modulation view
- larger scientific-style analysis panel where useful

### Effects

- delay
- reverb
- distortion
- equalisation
- resonator
- shimmer
- spectral blur
- grain delay
- additional creative effects as the product matures

## Data support

The fuller product should support a stronger data path than the demo.

This may include:
- richer asset libraries
- user import workflow
- external preprocessing support
- clearer source attribution
- source metadata viewing
- optional recent-data packs

## Preset and scene model

The product should distinguish among:
- preset
- node
- scene
- project

A preset defines a reusable track sound.

A node defines a saved morph position on a touch surface.

A scene defines a coordinated multi-track state.

A project contains the whole working session.

## Modes of use

### Live mode

Live mode should prioritise:
- immediate playability
- fast scene changes
- large touch interaction
- low visual friction
- stable recall

### Sound bank mode

Sound bank mode should prioritise:
- preset browsing
- source exploration
- patch refinement
- metadata visibility
- repeatable sound design

## Commercial readiness considerations

The fuller product should later address:
- licensing
- installer design
- analytics only if desired and ethically justified
- versioning
- release channels
- support workflow

## Product quality criteria

The fuller product should be judged by:
- sound identity
- stability
- responsiveness
- clarity of interaction
- strength of recall
- professional presentation
